# North Mini Code and Laguna XS.2 on SGLang v0.5.15

Focused single-user decode results for the repository's current SGLang v0.5.15 patch stack on two AMD
Radeon AI PRO R9700 GPUs.

## Configuration

- gfx1201, 32 GB per GPU, TP=2, ROCm 7.2, RCCL 2.27.7
- Conda environment `sglang-triton36-v0515`; source tree `/data/sgl-v0515`
- Triton attention and MoE; FP8 checkpoint weights and FP8 KV cache
- HIP graph capture at batch size 1
- Median TPOT converted with `decode tok/s = 1000 / TPOT_ms`

Structured measurements and run counts are in
[north-laguna-v0515-r9700-2026-07-12.json](north-laguna-v0515-r9700-2026-07-12.json).

## Retained changes

| Change | Isolated result | End-to-end disposition |
|---|---|---|
| Tuned R9700 MoE configurations | Paired kernels: about +5.1% Laguna, +11.2% North | Enabled for both models |
| Fused HIP sigmoid router | 0.0082 ms vs 0.028–0.032 ms; identical expert IDs | Enabled on HIP |
| Laguna BF16 router GEMV | 0.020 ms vs 0.064 ms; matching sampled IDs | Enabled for Laguna |
| Laguna BF16 attention collective | Avoids the defensive FP32 payload used by recurrent models | Scoped to Laguna |
| Triton RMSNorm and fused q/k norm | 5–5.5× and 4–8× kernel speedups | Enabled on supported HIP paths |
| Fused static-scale FP8 K/V stores | 1.77× / 2.56× / 2.57× / 6.36× at M=1/8/64/4096; byte-identical | Enabled for gfx1201 E4M3 NHD cache |

Custom all-reduce plus one-stage all-reduce was neutral. BF16 KV cache was neutral and reduced capacity.
A fused Laguna attention-gate prototype was faster in isolation but regressed the server curve and was
not retained. Laguna's `--swa-full-tokens-ratio 0.01` increased measured token capacity from 875,509 to
1,009,385 without a decode regression.

## Correctness and replay gates

- North: 34/36 text tests; coherent through 219,352 input tokens; tool call passed.
- Laguna: 34/36 text tests; capabilities 2/2; coherent through 220,277 input tokens; tool call passed.
- Router IDs matched their references, RMSNorm GPU comparisons passed, and fused stores were byte-identical.
- The current v0.5.15 series applied 56/56 patches, matched all 93 represented source paths, and passed 32/32 focused tests.
